2 E1 big picture 🟢 Foundational

E1 is the plain baseline: a single FP32 student model (MobileNetV3-Large + LR-ASPP) trained normally, on its own. It is the “control group” every later, fancier version is compared against.

Why keep the baseline pure? If E1 already had extra tricks, you couldn't tell how much each trick later adds. So E1 is deliberately kept simple — and this guide keeps “current E1” and “future stages” strictly separate.

🚫 E1 invariants — the rules this whole guide protects:

  • E1 is the FP32 baseline. No teacher, KD, CWD, QAT, or PTQ in E1.
  • num_classes = 116 · background_index = 0 · ignore_index = 255.
  • Image tensor [B,3,512,512] · mask [B,512,512] · logits [B,116,512,512].
  • Loss = CE + Dice, equal weight. Checkpoint metric = all-class validation mIoU.
  • Foreground / “disease-only” mIoU is provisional / secondary only.
  • Seed = 42. Augmentation is train-only; val/test are never augmented.
  • Dry-run = wiring only, not accuracy. Real run needs both --real-run and --confirm-real-run plus CUDA.
  • Checkpoints live outside the repo. Never commit outputs/logs/checkpoints. Never touch docs/reference/reference.pdf.
  • PLANTSEG_DATA_ROOT controls the dataset path.

3 Workflow map & how to read code safely 🟡 Core

Four layers work together:

  • Configs (configs/…) — settings only (numbers/names); no training logic.
  • Code (src/…) — the logic that reads the configs and does the work.
  • Smoke tests (scripts/smoke_*.py) — quick checks that each piece is wired right.
  • RunPod — the rented GPU where the real run finally happens.

Dry-run vs real run. A dry-run rehearses the whole loop tiny and fast (wiring only). A real run trains for real on a GPU and produces the model. You always pass the dry-run before the (gated) real run.

Two reading conventions you'll see: NEED_TO_CONFIRM = an honest “not decided yet, don't guess” placeholder; PROVISIONAL = a number reported but not the official headline.

🚫 Future-stage firewall. Teacher, distillation (E2/E3), and quantization (E4–E7) are not part of E1 and appear only in Section 17. Never treat them as current runnable work.

❓ Which layer holds settings only, with no training logic?

The config files (configs/…). The src/… code reads and executes those settings.

A. Executive verdict

6 Preprocessing & augmentation 🔴 Deep-dive

📄 src/data/transforms.py  +  configs/augment.py🔴 Deep-dive

What it's for: the preparation table — it resizes, pads, (for training) augments, and normalizes each image and its mask into the exact tensors the model expects, keeping the two perfectly aligned.

⬇️ In: one opened image + mask; augmentation params from configs/augment.py.
⬆️ Out: image [3,512,512] float32 (ImageNet-normalized) + mask [512,512] int64 (raw ids).
💬 Mental model: the kitchen prep table — trims every ingredient (image) and its recipe card (mask) to the same board size, cut identically, then plates them in the chef's required format.
Key shape rules: every sample becomes 512×512. Image → CHW float32; mask → HW int64. Masks resized with nearest (never blended); mask padding = 255; image padding = neutral mean color.
🚫 Locked — do not change: nearest for masks; pad mask with 255; ImageNet mean/std normalization; CHW image / HW mask; augmentation is train-only — val/test are never augmented; geometry always applies to image and mask together; color changes are image-only; no label remap.
⚠️ Common mistakes: resizing/rotating a mask with bilinear (invents fake labels); converting a mask to RGB; padding a mask with 0; changing size/normalization without changing the model; mixing up CHW and HWC; augmenting val/test.

❓ Why resize the mask with “nearest” and not smooth blending?

class LRASPPHead(nn.Module):
    def __init__(self, low_ch, high_ch, inter_ch, num_classes):
        self.high_proj = _ConvBNReLU(high_ch, inter_ch, 1)     # 1x1 Conv-BN-ReLU
        self.context   = nn.Sequential(AdaptiveAvgPool2d(1), Conv2d(high_ch, inter_ch, 1), Sigmoid())
        self.high_logits = nn.Conv2d(inter_ch, num_classes, 1)
        self.low_logits  = nn.Conv2d(low_ch,  num_classes, 1)
        self.ff_mul = FloatFunctional(); self.ff_add = FloatFunctional()   # quant-safe (plain in FP32)
    def forward(self, low, high) -> torch.Tensor:
        a = self.high_proj(high)                               # [B,inter,H16,W16]
        s = self.context(high)                                 # [B,inter,1,1]
        high_feat = self.ff_mul.mul(a, s)                      # attention
        high_os8  = F.interpolate(high_feat, size=low.shape[-2:], mode="bilinear", align_corners=False)
        return self.ff_add.add(self.high_logits(high_os8), self.low_logits(low))   # OS8 class logits

class SoftDiceLoss(nn.Module):
    def forward(self, logits, target) -> torch.Tensor:
        probs = F.softmax(logits, dim=1)                        # [B,C,H,W]
        valid = (target != self.ignore_index).unsqueeze(1).float()
        tgt = target.clone(); tgt[target == self.ignore_index] = 0    # placeholder; masked out below
        onehot = F.one_hot(tgt, num_classes).permute(0,3,1,2).float()
        probs = probs * valid; onehot = onehot * valid          # drop ignore pixels from BOTH
        dims = (0, 2, 3)
        inter = (probs * onehot).sum(dims); denom = probs.sum(dims) + onehot.sum(dims)
        dice = (2*inter + self.smooth) / (denom + self.smooth)
        present = onehot.sum(dims) > 0                           # classes present in this batch
        dice_sel = dice[present] if present.any() else dice
        return 1.0 - dice_sel.mean()

In: logits + target. Out: a scalar Dice loss. Correctness (deep): ✅

  • Soft & differentiable: Dice runs on softmax(logits) probabilities, not argmax, so it has a gradient.
  • Ignore-255 masked before counting: valid = (target != 255); both probs and onehot are multiplied by valid, so ignore pixels contribute nothing to intersection or denominator. The tgt[ignore]=0 placeholder is safe because those pixels are then zeroed by valid — they never add to class 0.
  • Per-class macro over present: inter/denom reduce over (B,H,W) → per-class [C]; dice_c = (2·inter_c + smooth)/(denom_c + smooth); present = valid-onehot-sum > 0 selects classes with ≥1 valid GT pixel in the batch; the mean is over present only (absent excluded) — matching the contract and the all-class-vs-foreground metric convention.
  • Degenerate guard: if no class is present (all-ignore batch), it falls back to the full dice vector (each ≈ smooth/smooth = 1.0 → loss ≈ 0); impossible in real training, benign.
  • Smoothing 1e-5 on numerator and denominator prevents div-by-zero and stabilizes tiny classes.

Efficiency: a [B,116,H,W] one-hot + masked reductions; no loops. Reproducibility: deterministic. Locked/safe: validity mask on both tensors, macro-over-present, smoothing. Beginner misread: thinking Dice has an ignore_index arg — it masks manually. Bug/risk: none (a valid target ≥ num_classes would make one_hot raise, but the dataset guarantees {0..115,255}).

What it's for: the scoreboard — turns predictions + the true mask into a confusion matrix and computes mIoU. A metric judges; it never updates the model.

E. Metric math walkthrough

  1. Predictions in. The caller computes pred = logits.argmax(1) (class id ∈ [0,115] per pixel); metrics.py receives class ids, not logits.
  2. Flatten. pred, target → 1D (same length; both derive from [B,512,512]).
  3. Drop ignore. valid = target != 255; keep pred[valid], target[valid] — ignore pixels removed before counting.
  4. Encode pairs. k = target·C + pred (unique per (t,p)).
  5. Count. bincount(k, minlength=C²) → reshape [C,C] = confusion matrix (rows=GT, cols=pred).
  6. Per-class terms. diag = intersection; row = GT total; col = pred total.
  7. Per-class IoU. diag / (row + col − diag) (clamped denominator).
  8. Presence. present = row > 0 (GT-present classes only); optional subset intersection.
  9. Macro mIoU. mean of iou[present]; NaN if none present.
  10. Selection. train_e1 sums cm across the whole val set, then computes all-class mIoU once → the checkpoint criterion; foreground mIoU is computed from the same cm and only logged.

Each step's failure mode is prevented: (3) prevents padding inflation; (5) minlength prevents shape errors; (7) clamp_min prevents div-by-zero; (8) prevents absent classes from corrupting the mean; (9) NaN is isolated to non-checkpoint paths on real val.

📄 src/seeds.py🟢 Foundational

What it's for: makes the project's “randomness” repeatable, so a run can be reproduced and two runs compared fairly.

⬇️ In: a seed number (here, 42).
⬆️ Out: nothing returned — it sets the whole program's randomness so results repeat.
💬 Mental model: fixing the same starting shuffle for every deck before the experiment — it doesn't make results better, just repeatable.
🚫 Locked — do not change: SEED = 42; seed Python + NumPy + PyTorch (CPU and all GPUs); cuDNN deterministic + benchmark off; call it first, before any data/model creation.
⚠️ Common mistakes: thinking a seed improves accuracy; seeding only one library; changing the seed casually (a different, non-comparable run); confusing deterministic with fast.

❓ Does using seed 42 make the model more accurate?

No. It makes the run reproducible/controlled; it says nothing about quality — a model can be reproducibly bad.

A smoke test pokes one piece to confirm it's connected right. They're cheap proof before you rent an expensive GPU.

🚫 A PASS means “wiring works,” NOT “model is accurate.” A few iterations on random/synthetic data can't make a good model.

18 Master glossary 🟢 Reference

tensor
A grid/cube of numbers the model does math on.
mask / target
A label grid where each pixel is an integer class id; the “answer key.”
logits
The model's raw per-class scores, before probabilities.
softmax / argmax
Softmax turns scores into probabilities; argmax picks the highest-scoring class per pixel.
loss
One number measuring how wrong the model is; training makes it smaller (E1 = CE + Dice).
metric / mIoU
A quality score that judges (doesn't train); mIoU = mean overlap across classes.
confusion matrix
A table of true-vs-predicted counts; mIoU is computed from it.
checkpoint
A saved model snapshot (weights + state); the “best” one has the highest all-class val mIoU.
ignore_index (255)
“Skip this pixel” — not a real class; excluded from loss and metrics.
CHW / HW
Axis orders: image = Channels-Height-Width; mask = Height-Width.
output stride (OS)
How much smaller a feature map is than the input (OS16 = 16× smaller).
augmentation
Random training-image changes (flip/rotate/crop/color); train-only.
normalize
Rescale pixels to a standard level (ImageNet mean/std) for stable training.
seed
A fixed number that makes randomness repeatable (here 42).
reproducibility
Getting the same result again by fixing seeds, versions, and data.
optimizer / SGD
The rule that updates weights from gradients (E1 uses SGD).
scheduler
Lowers the learning rate over time (E1 uses polynomial decay).
iteration vs epoch
Iteration = one batch/step; epoch = one full pass. E1 counts iterations.
dry-run
A tiny safe rehearsal that only checks wiring (CPU, random, no download).
real-run
The actual GPU training that produces the model (needs both flags + CUDA).
CUDA / GPU / CPU
CUDA lets code use a GPU (fast, rented, paid); the CPU is fine only for tiny checks.
pre-flight
The checklist completed before the real run.
PLANTSEG_DATA_ROOT
The environment variable that tells the code where the dataset is.
out-of-repo checkpoint
A saved model kept outside the project folder (never in version control).
FP32
Full 32-bit precision; E1 uses this. Quantization (later) shrinks it to 8-bit.
knowledge distillation / quantization
Future stages: teach a student from a teacher; shrink to INT8. Not in E1.

19 Quiz / review bank 🟢 Foundational

Test yourself across the whole path. Think first, then reveal.

❓ (Big picture) Is a “teacher model” part of E1?

No. E1 is the pure single-model FP32 baseline. Teacher + distillation are future stages.

❓ (Data) What decides a sample's split — the folders or the JSON files?

The folders (images/<split>/, annotations/<split>/). The JSON files are not used for split membership.

❓ (Preprocess) Which transforms apply to the mask, and which don't?

All geometric transforms (resize/pad/crop/rotate/flip) apply to image and mask together. Color transforms apply to the image only.

❓ (Model) Why must the output be 512×512?

So it matches the mask resolution and the loss can compare scores to labels pixel-for-pixel.

❓ (Loss) Do the ignore (255) pixels affect the loss?

No — they're excluded, so padding never pushes the model one way or another.

❓ (Metric) Why doesn't the metric train the model?

mIoU is based on hard argmax predictions and isn't a smooth signal for gradients; training uses the smooth loss. Metrics only judge.

❓ (Seeds) Why seed Python, NumPy, and PyTorch separately?

They're independent random systems; seeding one doesn't affect the others, and different parts of the pipeline use different ones.

❓ (Training) Where are checkpoints allowed to be saved?

Only outside the repo. The code refuses to save inside it, keeping large files out of version control.

❓ (Smoke) Does a smoke test train the model?

No. It only checks that a piece is wired correctly. PASS = wiring works, not accuracy.

❓ (RunPod) The most common launch failure?

Forgetting to set PLANTSEG_DATA_ROOT, so the loader can't find the data.
